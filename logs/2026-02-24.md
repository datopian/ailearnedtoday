### Karpathy on CLIs for AI Agents

Andrej Karpathy tweeted that CLIs are "super exciting precisely because they are a 'legacy' technology, which means AI agents can natively and easily use them, combine them, interact with them via the entire terminal toolkit." His example: ask your Claude/Codex agent to install the Polymarket CLI and query it. This validates the broader pattern we're seeing — products with great CLIs (Supabase, Vercel, Resend) are surging because AI coding agents default to them. Products without CLI/API access get skipped entirely.

### AgentMail — Email Inbox API for AI Agents

YC-backed service giving AI agents their own email inboxes via API. Register an agent, claim an @agentmail.to handle, and the agent can send/receive email autonomously. Related to SendClaw (autonomous email for AI agents at sendclaw.com) which Anu flagged earlier this month. ### The "AI-Agent Ready" Product Checklist

From today's Innovation Team Sync. The pattern that keeps coming up: Supabase surged in the last 6 months because AI agents default to it — great docs, CLI, and API mean vibe-coders' agents just pick it over SendGrid or Brevo without being asked. The practical test: open Claude Code, say "build a data portal with PortalJS," and see if it can find anything useful. If the agent bails because docs are missing, that's not just an AI problem — it's a human problem too. Takeaway: documentation, CLI, and API-first design are now table stakes, not nice-to-haves.

### DH — A Proposed Data Hub Command-Line Tool

Rufus proposed building a `DH` CLI for Data Hub — the default interface for both AI agents and human developers. Core idea: push a CSV or folder to Data Hub and get a live, versioned dataset with a nice viewer, stored on R2. No GitHub sync — push directly to storage, like what Flower Show's CLI already does. Separately, a **Data Presser** tool handles data preparation/wrangling (increasingly an AI skill-based task). Longer term: lightweight git-like versioning with content-addressable storage for transactional pushes and change logs, inspired by Hugging Face's Xet acquisition. MVP could be a 2–3 day experiment building against R2 from scratch.

### Publishing Pivot — Raw Logs to Substack

Strategy shift: we're no longer writing polished narrative blog posts. Instead, the daily logs from this repo (`logs/`) get posted directly to Substack as-is. Two Substack properties getting priority: **"What's Cooking"** (project updates, shipped features, demos) and **"AI Learned Today"** (this — daily tools, links, and ideas). YouTube videos are deprioritized (too slow, need approval), but YouTube Shorts continue — they're outperforming general videos (180+ views vs general).
