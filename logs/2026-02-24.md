### Proposal: DH — An AI-Native Data Hub Command-Line Tool

Rufus outlined the vision for `DH`, a new command-line tool for data publishing that is AI-native from the ground up. The core idea: instead of building a traditional GUI-first platform with an API bolted on, DH starts as a CLI that AI agents and human developers use equally. Key pillars include frictionless dataset publishing (push a CSV or Parquet file and get a hosted, versioned dataset), built-in schema inference, and an LLM-friendly metadata layer so agents can discover and query datasets conversationally. This aligns with the broader thesis that data publishing infrastructure should be designed for an AI-first world rather than retrofitted.

### Publishing Pivot — Prioritizing "What's Cooking" and "AI Learned Today"

Strategy shift from the Innovation Team Sync: we are deprioritizing polished, long-form blog posts. Going forward, the two Substack properties getting the most attention are **"What's Cooking"** (internal project updates, shipped features, demos) and **"AI Learned Today"** (daily log of tools, articles, and ideas encountered). The daily logs from this repo (`logs/`) will be published directly to Substack as-is — raw, heading-structured, with screenshots — rather than being rewritten into narrative essay form. This cuts publishing friction dramatically and keeps the content authentic. The goal is consistent, frequent output over polished infrequency.
