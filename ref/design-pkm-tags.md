# Design & PKM Tags

Lightweight taxonomy for design tools and personal knowledge management (PKM) patterns used in this knowledge base.

## Tags

### `designapps`

Tools and patterns focused on **design workflows**, especially landing pages, UI systems, and visual exploration. Often small, opinionated tools rather than full design suites.

Examples:
- [[aura-build]] – AI website builder that generates DESIGN.md from any URL
- [[stitch-google]] – AI-native design canvas with DESIGN.md and MCP integration
- [[variant]] – AI design tool for generating high-quality UI variants
- [[whizz-ai-design-prompts]] – Curated AI prompts for UI design
- [[tambo-ai]] – Generative UI SDK for React
- [[mockdown-design]] – ASCII wireframe editor for fast layout iteration
- [[huemint]] – ML-based color palette generator
- [[typevibe]] – Vibe-focused typography playground for landing pages

### `pkmpatterns`

Patterns and tools for **personal knowledge management** and long-term memory for AI and humans. Emphasis on structure, recall, and multi-session continuity.

Examples:
- [[mempalace]] – Local-first AI memory palace with AAAK compression and MCP tools
- [[artem-grep-is-dead]] – Memory system treating logs as searchable knowledge
- [[qmd]] – Local search engine for markdown notes (BM25 + vectors + re-ranking)
- [[kris-puckett-obsidian-ai]] – Example of AI-augmented Obsidian setup
- [[economists-ai-adoption]] – Early examples of academics using AI as research memory

## Usage

- Use `designapps` tag for entries that are **primarily about design tools or design workflows**.
- Use `pkmpatterns` tag for entries that are **primarily about memory, PKM, or long-term context**.

This file is intentionally minimal — it’s a living index we can expand as new clusters emerge (e.g., `agentinfra`, `aipolicy`).
